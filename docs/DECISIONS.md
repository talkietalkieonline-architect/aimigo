# AIMIGO — Ключевые решения

---

## D001: Ребрендинг Talkie-Talkie → Aimigo
**Дата:** Июнь 2025
**Решение:** Проект переименован. Строим с нуля на новом стеке.
**Причина:** Старый проект на vanilla JS стал неуправляемым (5500+ строк в одном файле).

## D002: Технологический стек
**Дата:** Июнь 2025
**Решение:** Next.js + FastAPI + PostgreSQL + Redis
**Причина:** Масштабирование по городам и странам, собственные GPU-фермы, микросервисы в будущем.

## D003: Начинаем с нуля, а не рефакторим
**Дата:** Июнь 2025
**Решение:** Новый проект с чистой архитектурой. Старый код (v0.39) — как референс.
**Причина:** Разбиение старого index.html на модули привело к поломкам. Проще построить правильно.

## D004: Система документации
**Дата:** Июнь 2025
**Решение:** 4 файла документации (ТЗ, Прогресс, Архитектура, Решения) обновляются каждую сессию.
**Причина:** AI-ассистент теряет контекст между сессиями. Документация = память проекта.

## D005: Фазовая разработка
**Дата:** Июнь 2025
**Решение:** 3 фазы — MVP → Голос и режимы → Масштаб
**Причина:** Не утонуть в масштабе. Сначала рабочий продукт, потом расширение.

## D006: Игнорировать "Мир агентов"
**Дата:** Июнь 2025
**Решение:** Концепция "Мир агентов" из старого проекта — ошибочное направление.
**Причина:** Решение владельца проекта. Используем только "Город агентов" (каталог).

## D007: Voice-first UX
**Дата:** Июнь 2025 (Сессия 5)
**Решение:** Основной способ ввода — голос. Текстовое поле скрыто по умолчанию, появляется по кнопке ⌨️.
**Причина:** Платформа идёт к говорящим аватарам с мимикой. Голос — основной UX. Клавиатура на телефоне закрывает треть экрана.

## D008: Кнопка Mute убрана, функция на микрофоне
**Дата:** Июнь 2025 (Сессия 5)
**Решение:** Короткое/длинное нажатие на микрофон = вкл/выкл/always-on/mute.
**Причина:** Освобождаем место для кнопки переключателя ввода.

## D009: Пузыри — пользователь СЛЕВА, агенты СПРАВА
**Дата:** Июнь 2025 (Сессия 5)
**Решение:** Пользователь слева (аватар «Я»), агенты и собеседники справа (аватар с буквой).
**Причина:** Решение владельца проекта. Агенты справа — ближе к правой панели «Команда эфира».

## D010: Бизнес-визия задокументирована
**Дата:** Июнь 2025 (Сессия 5)
**Решение:** Создан BUSINESS_VISION.md — монетизация, аудитории, режимы, коммерческий блок.
**Причина:** AI-ассистент должен понимать полную картину продукта в каждой новой сессии.

## D011: Graceful fallback — фронт работает без бэкенда
**Дата:** Июнь 2025 (Сессия 6)
**Решение:** Все API-вызовы обёрнуты в try/catch. При недоступном бэкенде фронт использует локальные данные (хардкод-агенты, оффлайн-чат, локальная SMS-верификация).
**Причина:** Можно разрабатывать UI без запущенного сервера. Когда Docker поднят — всё заработает автоматически.

## D012: React Context для авторизации (не Redux / Zustand)
**Дата:** Июнь 2025 (Сессия 6)
**Решение:** Единый AuthContext вместо внешнего стейт-менеджера.
**Причина:** Минимум зависимостей на MVP. Авторизация — простой стейт (user + isLoggedIn). Переход на Zustand при усложнении.

## D013: Хуки вместо inline-логики в page.tsx
**Дата:** Июнь 2025 (Сессия 6)
**Решение:** useChat, useAgents — отдельные хуки в /hooks/.
**Причина:** page.tsx был 150+ строк с inline-логикой чата и демо-данными. Теперь чистый компонент композиции.
