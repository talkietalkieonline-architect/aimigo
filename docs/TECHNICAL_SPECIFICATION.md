# AIMIGO — Техническое задание v1.0

---

## 1. ОБЩЕЕ ОПИСАНИЕ

**Aimigo** — экосистема, объединяющая пользователей и бизнес через голосовой Коммуникатор с ИИ-агентами. Платформа где пользователь общается голосом с персональным Дворецким, подключает агентов бизнеса, друзей, и переключает режимы работы под задачи.

**Слоган:** AI-First

**Платформа:** Web (далее — мобильные приложения)

**Предыдущее название:** Talkie-Talkie Online

---

## 2. ТЕХНОЛОГИЧЕСКИЙ СТЕК

### Фронтенд
- React + TypeScript
- Next.js (SSR, роутинг, PWA)
- Tailwind CSS + CSS-переменные (темы/пресеты)
- WebSocket (реалтайм чат)
- WebRTC (голосовое общение)

### Бэкенд
- FastAPI (Python, модульная архитектура)
- PostgreSQL (основная БД)
- Redis (сессии, кэш, pub/sub)
- Celery + Redis (очереди задач)
- PostGIS (геолокация для режима Прогулка)

### Инфраструктура
- Docker (контейнеризация)
- Kubernetes (масштабирование по городам)
- S3 (хранение файлов, медиа, голос)
- Nginx (API Gateway)

### Масштабирование
- Архитектура микросервисов (Auth, Chat, Agents, Voice, Geo, Shop)
- Мультигородская система (первый город — Санкт-Петербург)
- Подготовка к собственным GPU-фермам (генерация голоса, видео)

## 3. ФЛОУ ПОЛЬЗОВАТЕЛЯ

### 3.1. Вход в систему

**Регистрация (первый раз):**
1. Заставка (лого Aimigo, анимация)
2. Номер телефона → SMS-код подтверждения
3. Создание пароля
4. Попадает в Коммуникатор
5. Дворецкий приветствует и предлагает мини-тур

**Вход (повторный):**
1. Номер + пароль (без SMS)
2. Сессия живёт ~30 дней
3. Дворецкий приветствует: «Привет, рад снова!»

## 4. КОММУНИКАТОР — ГЛАВНЫЙ ЭКРАН

### 4.1. Структура экрана

```
┌─────────────────────────────────────────────┐
│  AIMIGO                          AI-First   │
│  [ЭФИР ← бегущая строка новостей →]        │
├──┬──────────────────────────────────────┬───┤
│Л │                                      │ П │
│Е │         ЦЕНТРАЛЬНОЕ ОКНО             │ Р │
│В │         (чат / контент)              │ А │
│А │                                      │ В │
│Я │                                      │ А │
│  │                                      │ Я │
├──┴──────────────────────────────────────┴───┤
│ [Настройки] [Mute] [МИКРОФОН] [Контакты] [Агенты] │
└─────────────────────────────────────────────┘
```

### 4.2. Верхняя часть
- Логотип AIMIGO + слоган AI-First
- Бегущая строка "ЭФИР" — активируется при добавлении Агента новостей
  - Кликабельная (нажал → подробнее)
  - Озвучивается Агентом новостей
  - Источники: пробки, погода, МЧС, акции, письма

### 4.3. Центральное окно

Содержимое зависит от режима:
- Общение → Чат (пузыри + медиа)
- Прогулка → Чат + геолокация
- Обучение → Чат (лекция)
- Покупки → Плитки магазинов (маркетплейс + поиск)
- Работа → Чат (корпоративное пространство)

**Чат — правила отображения:**
- Сообщения пользователя — пузыри справа
- Сообщения собеседников (агенты + пользователи) — пузыри слева
- У каждого собеседника свой цвет пузыря + маленький аватар рядом
- Пузыри содержат: текст, картинки, видео, файлы
- Поток снизу вверх (новые внизу)

### 4.4. Нижняя панель — 5 кнопок

1. **Настройки** — Открывает Центр Управления
2. **Mute** — Выключить входящий звук (не слышать собеседника)
3. **Микрофон** (центр, большая) — 1-е нажатие = вкл (нон-стоп), 2-е = выкл
4. **Мои контакты** — Список друзей в сервисе
5. **Мои агенты** — Подборка агентов пользователя

### 4.5. Левая панель (выдвижная)

Открывается нажатием на золотой язычок слева.
Закрывается после выбора или автоматически через ~4 сек.
Наплывает поверх чата (overlay).

**Верхний блок — РЕЖИМЫ (скролл):**
1. Общение (по умолчанию)
2. Прогулка
3. Обучение
4. Покупки
5. Работа
(Список расширяемый)

**Нижний блок — КОМНАТЫ (зависят от режима, скролл):**
- Общение → Общая комната, Приватные, + Создать
- Покупки → Магазины
- Обучение → Классы / Лектории
- Прогулка → В разработке
- Работа → В разработке

### 4.6. Правая панель (выдвижная)

Открывается нажатием на золотой язычок справа.
Закрывается автоматически через ~4 сек.
Наплывает поверх чата (overlay).

**Содержимое:**
- Дворецкий — закреплён сверху ВСЕГДА
  - Короткое нажатие → выключить (ч/б, неактивен но виден)
- Слайдер участников — агенты и пользователи в разговоре
  - Короткое нажатие → выключить (ч/б)
  - Длинное нажатие → убрать из разговора
  - Аватар мерцает когда агент говорит

### 4.7. Адаптивность
- Мобилка и десктоп — одинаковый принцип (панели скрыты)
- Будущее: опция «Закрепить панели» для широких экранов
- Фуллскрин — тройное нажатие

## 5. ДВОРЕЦКИЙ

### 5.1. Описание
Персональный умный помощник. Всегда присутствует. Растёт и обучается вместе с пользователем.

### 5.2. Возможности
- Отвечает на вопросы о сервисе
- Ищет агентов в Городе и подключает к разговору
- Сравнивает агентов по запросу
- Записывает, напоминает
- Рассказывает о погоде
- Рекомендует полезных агентов
- В режиме Прогулка — связующий с агентами-зазывалами

### 5.3. Поведение в комнатах
- Общая комната → Активен, общается
- Приватная комната → Автоматически отключается, но виден (ч/б)
- Разговор с агентом → Не участвует, но слушает и запоминает

### 5.4. Кастомизация
Имя, пол, возраст, модель общения, тип общения, характер, настройки голоса

---

## 6. ГОЛОСОВОЕ ОБЩЕНИЕ

### 6.1. Основной принцип
Голос — основной способ общения. Чат на экране — дублирование.

### 6.2. Механика
- Пользователь говорит → слова появляются в чате
- Агент отвечает голосом → текст отображается в чате
- Микрофон: 1-е нажатие = вкл (нон-стоп), 2-е = выкл
- Mute: выключить входящий звук
- Если микрофон выключен — агент отвечает только текстом

### 6.3. Мультиагентный разговор
- Обращение к агенту по имени
- Дворецкий подключает агентов по запросу
- Дворецкий отходит но слушает и запоминает

### 6.4. Будущее
- Агент на экране с мимикой и эмоциями
- Смена облика и одежды

## 7. ГОРОД АГЕНТОВ

### 7.1. Описание
Каталог всех агентов. Доступ: кнопка «Мои агенты» → «Город Агентов», или голосом через Дворецкого.

### 7.2. Интерфейс
- Поиск (по имени, домену, роли)
- Фильтры по профессии (Консультант, Продавец, Психолог, Тренер, Аналитик, Ассистент, Собеседник)
- Фильтры по типу (Все, Жители, Бизнес, Системные)
- Счётчики (Всего, Бизнес, Жители)

### 7.3. Карточка агента в списке
- Кружок с аватаром
- Имя
- Профессия
- Компания / бренд
- Метка «уже у тебя» если добавлен

### 7.4. Меню агента (короткое нажатие)
- Смотреть презентацию (видео-визитка 15-30 сек)
- Добавить в Мои агенты (или «Уже у тебя в избранном»)
- Начать чат
- Оценить агента
- Пожаловаться

### 7.5. Aimigo Links
- Уникальная ссылка каждого агента (aimigo.com/a/tim-adidas)
- Для наружной рекламы, соцсетей, визиток
- У пользователей тоже есть свой Aimigo Link

## 10. РЕЖИМЫ РАБОТЫ

### 10.1. Общение (по умолчанию)
- Комнаты: Общая, Приватные, + Создать
- Центр: Чат с пузырями и медиа
- Дворецкий: Активен в Общей, отключён в Приватной
- Анимация: Медленные частицы

### 10.2. Прогулка
- Комнаты: В разработке
- Анимация: Радар
- Агенты-зазывалы привязаны к геоточкам
- Дворецкий — связующий между пользователем и зазывалами

### 10.3. Обучение
- Комнаты: Классы / Лектории
- Анимация: Лава-лампа
- Лекции по расписанию, вопросы лектору
- Дворецкий напоминает о записи
- Режим прослушивания (чат скрыт, только фон + голос)

### 10.4. Покупки
- Комнаты: Магазины
- Центр: Плитки магазинов + поиск
- Анимация: Сканер
- Два уровня: Агент-консультант (базовый) / Магазин с витриной (премиум)

### 10.5. Работа
- Корпоративное пространство
- Компания создаёт внутренних агентов
- Доступ только для сотрудников
- Анимация: В разработке

## 18. АДМИНКА — MISSION CONTROL

### Доступ
Отдельный защищённый вход. Кнопка ADMIN убирается из публичного интерфейса.

### Уровни
Глобальный → Город → Сущности внутри города

### Разделы
- Города — список, создание, статистика
- Пользователи — все, подписчики, аналитика по каждому
- Агенты — все, расходы генерации, API-ключи, модели, голос
- Бизнес — кабинеты, агенты, оплаты
- Аналитика — чаты, действия, воронки
- Система — нагрузки, мощности, мониторинг
- Безопасность — защита, логи, блокировки

### Суперагенты
- Агент-слушатель — мониторит работу агентов
- Агент-наблюдатель — следит за системой

### Масштабирование
Первый город: Санкт-Петербург. Создать город → наполнить → запустить.
